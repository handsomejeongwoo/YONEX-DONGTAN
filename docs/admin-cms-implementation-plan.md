# 요넥스 동탄점 관리자 CMS 구현 계획

## 목표

홍승인 대표님이 홈페이지의 핵심 콘텐츠를 직접 관리할 수 있도록 관리자 페이지를 만들고, 현재 하드코딩된 홈페이지를 데이터 기반 UI로 전환한다.

관리 대상은 배너, 추천 상품, 유튜브 영상, 인스타 릴스, 대회 이력, 매장 정보다. 대표님이 링크와 이미지, 문구를 수정하면 홈페이지에 반영되는 운영 구조를 목표로 한다.

## 전체 진행 순서

```text
1. 콘텐츠 구조 및 DB 스키마 설계
2. 기존 프론트를 DB 기반 UI로 전환
3. 관리자 로그인 및 대시보드 구현
4. 로컬 저장소로 CRUD 전체 테스트
5. Cloudflare D1/R2/Workers 연결
6. 실서비스 환경 테스트 및 대표님 전달
```

---

## 1. 콘텐츠 모델 및 초기 데이터 준비

현재 홈페이지에 들어간 데이터를 아래 콘텐츠 단위로 분리한다. 먼저 지금의 공개 정보를 초기 데이터로 넣고, 이후 관리자에서 수정할 수 있게 한다.

### 배너 (`banners`)

- 배너 이미지
- 컬러 테마: `blue` / `green` / `white`
- 카테고리 라벨
- 제목 / 설명
- 버튼 문구
- 이동 링크
- 이미지 위치: `left` / `center` / `right`
- 노출 순서
- 공개 여부
- 게시 시작일 / 종료일 (선택)

### 추천 상품 (`featured_products`)

- 상품명
- 상품 이미지
- 정가
- 판매가
- 할인율: 정가와 판매가를 기준으로 자동 계산
- 추천 문구
- 배지: `NEW` / `PICK` / `SALE`
- 스마트스토어 상품 URL
- 노출 순서
- 공개 여부

> 가격은 스마트스토어에서 변동될 수 있으므로, 공개 페이지에는 `스마트스토어에서 최종 가격 확인` 문구를 함께 둔다.

### 유튜브 영상 (`youtube_videos`)

- YouTube URL
- 영상 ID
- 영상 제목
- 썸네일 URL
- 카테고리: 장비 리뷰 / 경기 영상 / 쇼츠 등
- 노출 순서
- 공개 여부

YouTube URL 입력 후 영상 ID를 추출하고, 제목과 썸네일을 자동으로 가져온다. 자동 불러오기에 실패할 때를 대비해 제목과 썸네일을 직접 수정하는 기능도 둔다.

### 인스타 릴스 (`instagram_reels`)

- 릴스 URL
- 썸네일 이미지
- 제목 또는 짧은 문구
- 노출 순서
- 공개 여부

릴스 원본 영상은 저장하지 않는다. 썸네일을 클릭하면 해당 인스타 릴스로 이동한다.

### 대회 이력 (`achievements`)

- 연도
- 대회명
- 종목
- 파트너
- 결과: 우승 / 준우승 / 3위
- 상단 고정 여부
- 노출 순서
- 공개 여부

### 매장 정보 및 공지 (`store_settings`)

- 주소 / 전화번호 / 영업시간 / 휴무일
- 스마트스토어 / 유튜브 / 인스타 / 네이버 지도 링크
- 상단 공지 문구
- 공지 노출 시작일 / 종료일

## 2. 홈페이지 프론트 전환

현재 하드코딩된 텍스트와 카드 데이터를 콘텐츠 저장소에서 읽도록 변경한다.

### 홈 개편

- 헤더: `홈 · 동탄점 소개 · 사장님 소개 · 영상 · 오시는 길 · 스마트스토어`
- 기존 홈의 매장 소개는 별도 `동탄점 소개` 페이지 또는 섹션으로 분리
- 홈 최상단에는 자동 전환 배너 배치
- 추천 상품, 유튜브, 릴스, 대회 이력은 관리 데이터에 따라 렌더링
- 비공개 처리한 콘텐츠는 홈페이지에 표시하지 않음

### 반응형 배너 원칙

대표님이 모바일용 이미지를 별도로 제작하지 않아도 자연스럽게 보이도록 구성한다.

- 데스크톱: 텍스트와 이미지를 좌우로 배치
- 모바일: 텍스트를 위, 이미지를 아래로 세로 배치
- 배너 문구는 이미지에 포함하지 않고 HTML 텍스트로 렌더링
- 이미지는 원본 비율을 우선 보존하고, 필요한 경우 선택한 이미지 위치 기준으로 가볍게 크롭
- 권장 업로드: JPG/WebP, 3~5MB 이하

## 3. 관리자 페이지 구현

### 진입과 인증

- `/admin`: 로그인 상태가 아니면 로그인 화면으로 이동
- `/admin/login`: 흰 배경, 요넥스 동탄점 로고, 관리자 코드 입력창 1개
- 푸터의 동탄점 로고 5회 클릭 시 `/admin/login`으로 이동
- 관리자 코드는 `.env.local`의 `ADMIN_ACCESS_CODE`로 관리
- 로그인 성공 시 서명된 세션 쿠키 발급
- 로그아웃 및 비정상 접근 차단

> 푸터 로고 5회 클릭은 진입 편의 기능일 뿐 보안 기능이 아니다. 실제 권한 검증은 서버 측 코드와 세션으로 처리한다.

### 관리자 메뉴

```text
대시보드
배너 관리
추천 상품
유튜브 영상
인스타 릴스
대회 이력
매장 정보
```

### 공통 관리 기능

- 등록 / 수정 / 삭제
- 임시저장 / 공개 여부
- 노출 순서 변경
- 홈페이지 미리보기
- 이미지 업로드 및 미리보기
- 삭제 대신 비공개 처리 지원
- JSON 또는 CSV 백업 내보내기 (2차)

### UI 원칙

- shadcn/ui를 사용해 테이블, 입력 폼, 다이얼로그, 토스트 중심으로 빠르게 구현
- 운영 화면이므로 시각 효과보다 입력 오류 방지와 명확한 상태 표시를 우선
- 대표님이 모바일에서도 간단한 수정은 할 수 있게 반응형 지원

## 4. 로컬 저장소 및 동작 테스트

### 로컬 단계

- 초기에는 `data/content.json`을 저장소로 사용
- 콘텐츠 접근 코드를 `repository` 레이어로 분리
- 관리자 UI는 저장소 구현을 모르도록 구성
- 이후 Cloudflare D1으로 교체할 때 UI와 API 형태를 유지

### 검증 항목

- 관리자 코드 로그인 / 로그아웃 / 잘못된 코드 처리
- 로그인하지 않은 상태의 `/admin` 접근 차단
- 배너, 상품, 영상, 릴스, 대회 이력의 등록·수정·비공개·삭제
- 상품 할인율 자동 계산
- 유튜브 URL 처리 및 수동 입력 대체 흐름
- 콘텐츠 노출 순서 변경
- 변경 사항이 홈페이지에 바로 반영되는지 확인
- 모바일 360px, 태블릿, 데스크톱 반응형 확인
- TypeScript 검사 및 프로덕션 빌드 검사

## 5. Cloudflare 전환 및 배포

로컬 동작이 확정된 후 아래 순서로 운영 환경에 연결한다.

```text
data/content.json  → Cloudflare D1
로컬 업로드 이미지 → Cloudflare R2
.env.local         → Cloudflare Secrets
Next.js            → Cloudflare Workers (OpenNext)
```

### 운영 환경 원칙

- D1: 텍스트, 링크, 상품·영상·대회 데이터
- R2: 배너, 상품 이미지, 릴스 썸네일
- Workers: 공개 페이지와 `/admin` API·세션 처리
- 외부 영상: 유튜브/인스타 원본은 저장하지 않고 링크만 사용
- 스마트스토어: 크롤링하지 않고 대표님이 지정한 상품 링크를 노출

### 배포 전 최종 확인

- 실제 관리자 코드와 Cloudflare Secret 설정
- 도메인 연결
- 모바일 실기기에서 관리자 등록 및 공개 페이지 반영 확인
- 배너 링크, 스마트스토어 링크, 지도, YouTube, Instagram 연결 확인
- 대표님에게 관리자 사용 방법 1페이지 안내 전달

- 스마트스토어 공식 연동
